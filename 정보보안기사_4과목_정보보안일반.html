Chapter 01 · POINT 011

정보보호 목표
— CIA 3요소, 보안통제, FDS

정보보안의 모든 개념은 CIA 3요소 위에서 출발해. 무엇을 왜 보호하는지 이해하면 이후 나오는 모든 보안 기술이 어떤 목적을 위한 것인지 자연스럽게 연결돼.
정보보호의 목표 — CIA 3요소 필기+실기
목표설명위협 예시
기밀성
(Confidentiality)		 정보가 허가되지 않은 사용자(조직)에게 노출되지 않는 것을 보장하는 보안 원칙 도청, 스니핑, 정보 유출
무결성
(Integrity)		 정보가 권한이 없는 사용자의 의의적 또는 비의의적인 접근에 의해 변경되지 않는 것을 보장하는 보안 원칙 데이터 변조, 위·변조 공격
가용성
(Availability)		 인가된 사용자(조직)가 정보시스템의 데이터 또는 자원을 필요로 할 때 부당한 지체 없이 원하는 객체 또는 자원을 접근하고 사용할 수 있는 것을 보장하는 보안 원칙 DoS/DDoS 공격, 랜섬웨어
💡
추가 보안 목표
• 부인방지(Non-Repudiation): 송신자가 메시지를 보낸 사실을 부인하거나, 수신자가 받은 사실을 부인하지 못하게 함 → 전자서명으로 구현
• 인증(Authentication): 사용자가 주장하는 신원이 맞는지 확인
보안통제 — 시점별 통제의 종류 필기+실기
통제 유형설명사례
예방 통제
(Preventive Control)		 바람직하지 못한 사건이 발생하는 것을 피하기 위해 사용되는 통제 담장, 자물쇠, 보안 경비원, 백신, 직무분리, 암호화, 방화벽
탐지 통제
(Detective Control)		 발생된 사건을 식별하기 위해 사용 CCTV, 보안 감사, 감사로그, 침입탐지, 경보
교정 통제
(Corrective Control)		 발생된 사건을 교정하기 위해 사용 백신 소프트웨어
복구 통제
(Recovery Control)		 자원과 능력을 복구하기 위해 사용 백업과 복구
FDS (Fraud Detection System) 필기+실기
🔍
FDS는 전자금융거래에서 사용되는 단말정보, 접속로그, 거래정보 등을 분석하여 금전 및 사적인 이득을 취하기 위해서 발생하는 각종 부정 거래행위를 탐지 및 예방한다.
Chapter 02 · POINT 012

사용자 인증
— 악성코드, 정보 접근단계, 인증 방식, 생체인증

악성코드 종류 필기+실기
종류설명
논리폭탄
(Logic Bomb)		 특정 조건이 발생할 때 실행되는 악성코드. 특정 조건이 발생하지 않으면 악성코드로 작동하지 않기 때문에 탐지하기 어려움
키로거
(Key-Logger)		 사용자가 입력하는 정보를 갈취하는 악성코드. 윈도우 후킹(Hooking)을 통해서 키보드 입력정보를 획득
트로이목마
(Trojan)		 자기복제 능력이 없음. 시스템 정보, 개인정보 등을 유출하거나 특정 시스템 파일을 삭제
백도어
(Backdoor, Trapdoor)		 인증과정을 우회해서 시스템에 접근할 수 있도록 열어둔 뒷문을 통해 원격 조정하여 시스템을 장악
정보의 접근단계 필기
단계설명예시
식별
(Identification)		 사용자가 시스템에 본인이 누구인지 밝히는 행위 사용자 ID 입력
인증
(Authentication)		 정보나 시스템에 접근하려는 사용자가 본인임을 확인하는 단계. 인증을 주장하는 정당한 사용자임을 확인 Password, 스마트카드, 생체인증
인가
(Authorization)		 인증을 거친 사용자에게 허용된 접근 권한과 사용 범위를 부여하는 단계 파일 읽기/쓰기 권한 부여
🧠 순서 암기
식별 → 인증 → 인가 (나는 누구 → 맞는지 확인 → 권한 줌)
인증 방식의 분류 필기
구분설명기반종류
Type I 인증Something you know지식Password, Pin, Passphrase
Type II 인증Something you have소유Smart Card, Tokens, Key
Type III 인증Something you are존재(생체)홍채, 지문, 정맥
Type IV 인증Something you do행동음성, 서명, Keystroke Dynamics
💡
다중인증(MFA): 두 가지 이상의 Type을 결합. 예: ATM 카드(Type II) + PIN(Type I) → 보안 강화
생체인증 오류율 필기
용어설명
FRR
(False Reject Rate, Type I Error)		 잘못된 거부율. 편의성 관점: 정상적인 사람을 거부함
FAR
(False Acceptance Rate, Type II Error)		 잘못된 승인율. 보안 관점: 비인가자를 정상 인가자로 받아들임
CER / ERR
(Crossover Error Rate, Equal Error Rate)		 FRR과 FAR이 교차되는 지점. 효율성 및 생체인증의 척도. CER이 낮을수록 좋은 시스템
📊
FRR ↑ = 보안 강화, 불편함 증가
FAR ↑ = 편의성 증가, 보안 약화
CER(ERR) = 두 오류율이 같아지는 지점 → 생체인증 시스템의 종합 성능 지표

Kerberos
— 인증 과정, 버전별 특징

Kerberos는 네트워크 환경에서 티켓(Ticket) 기반으로 인증하는 프로토콜이야. 패스워드를 네트워크로 직접 전송하지 않아서 도청 공격에 강해. 기업 내부망(Active Directory)에서 많이 사용돼.
Kerberos 인증 과정 필기
단계방향내용
① AS 요청클라이언트 → AS사용자 ID(평문)를 AS에 요청
② TGS 세션키 응답AS → 클라이언트TGS 세션키 + TGT(Ticket Granting Ticket) 발급
③ TGT로 티켓 요청클라이언트 → TGSTGS 세션키와 TGT로 서비스 티켓 요청
④ SS 세션키 응답TGS → 클라이언트서비스 서버(SS) 세션키 발급
⑤ SS 세션키로 접속클라이언트 → SS발급받은 세션키로 서비스 서버에 접속

주소IP 주소 사용다른 형식의 주소 사용 가능
메시지 순서메시지 바이트 순서 표시 고정ASN.1과 BER 인코딩 규칙 표준 사용

인증발송인증발송 지원 안 함인증발송을 지원함

패스워드 공격패스워드 추측공격 가능사전 인증기능으로 패스워드 추측 공격이 더 어려워짐
Chapter 04 · POINT 013

접근통제
— MAC/DAC/RBAC 비교, 접근통제 모델

접근통제는 "누가, 어떤 자원에, 어떤 방식으로 접근할 수 있는지"를 결정하는 메커니즘이야. 세 가지 방식(MAC/DAC/RBAC)의 차이를 이해하면 보안 등급, 정책 설계와 자연스럽게 연결돼.
접근통제 기술 비교 — MAC / DAC / RBAC 필기
항목MAC (강제적 접근통제)DAC (임의적 접근통제)RBAC (역할기반 접근통제)
권한부여자System(관리자)Data Owner(소유자가 관리)Central Authority
접근여부 결정기준Security LabelIdentityRole
오렌지 북BCC
장점안전/중앙 집중 관리유연, 구현 용이관리 용이
단점구현/운영 어려움, 높은 비용트로이목마, ID 도용문제유연한 통제 어려움
🧠 핵심 구분
MAC: 시스템(관리자)이 보안 등급 라벨로 강제 통제 → 군사 환경
DAC: 데이터 소유자가 직접 권한 설정 → 일반 OS 파일 시스템
RBAC: 역할(Role)에 권한을 부여 → 기업 환경 (과장=결재권한, 사원=조회권한)
접근통제 모델 필기
🔐
Bell-Lapadula 모델 — 기밀성 모델
높은 등급의 정보가 낮은 레벨로 유출되는 것을 통제하는 모델. 군사 보안 모델의 기반.

Bell-Lapadula 보안규칙
① No Read-Up (NRU or ss-property) — 단순 보안 규칙
  주체는 자신보다 높은 등급의 객체를 읽을 수 없음
  주체의 취급인가가 객체의 비밀 등급보다 같거나 높아야 그 객체를 읽을 수 있음

② No Write-Down (NWD or *-property) = Confinement property — *(스타) 보안 규칙
  주체는 자신보다 낮은 등급의 객체에 정보를 쓸 수 없음
  주체의 취급인가가 객체의 비밀 등급보다 낮거나 같은 경우에 그 객체를 주체가 기록할 수 있음

③ Strong *-property
  더욱 강화된 모델로 주체는 자신과 등급이 다른 객체에 대해 읽거나 쓸 수 없음
모델특징
Bell-Lapadula 기밀성 모델. 높은 등급의 정보가 낮은 레벨로 유출되는 것을 통제. No Read-Up, No Write-Down
Biba 모델 Bell-Lapadula 모델의 단점인 무결성을 보장하는 모델. 주체에 의한 객체 접근의 항목으로 무결성을 다룸. No Read-Down, No Write-Up
Clark and Wilson
(클락 윌슨 모델)		 무결성 중심의 상업용으로 설계. Application의 보안 요구사항을 다룸. 정보의 특성에 따라 비밀 노출 방지보다 자료의 변조 방지가 더 중요한 경우에 사용
만리장성 모델
(Chinese Wall = Brewer-Nash)		 서로 상충 관계에 있는 객체 간의 정보 접근을 통제하는 모델 (이익의 상충 금지). 상업적으로 기밀성 정책을 따름
Chapter 05 · POINT 014

전자서명
— 과정, 종류, PKI

전자서명은 "이 문서를 내가 작성했다"는 것을 증명하는 디지털 기술이야. 종이에 도장 찍는 것과 같은 역할인데, 위조가 불가능하도록 암호학적으로 구현돼. 부인방지(Non-Repudiation)를 제공한다는 게 핵심이야.
전자서명 과정 필기+실기
구분키(Key)설명
송신자 개인키 전자서명 수행 — 내가 개인키로 서명 (내 도장)
공개키 전자서명 확인 — 상대방이 내 공개키로 서명 검증
수신자 개인키 복호화 수행 — 내 개인키로 암호화된 내용 복호화
공개키 암호화 수행 — 상대방의 공개키로 암호화해서 전송
💡
핵심 정리
암호화: 수신자의 공개키로 암호화 → 수신자의 개인키로 복호화 (기밀성)
전자서명: 송신자의 개인키로 서명 → 송신자의 공개키로 검증 (부인방지, 무결성)
전자서명 종류 필기+실기
종류특징
① RSA 전자서명 RSA 암호화 알고리즘을 사용해서 메시지에 대해 전자서명. RSA 방식은 암호화와 복호화를 하는 경우 송신자의 개인키와 공개키를 사용. 소인수 분해의 어려움을 기반으로 하는 전자서명
② ElGamal 전자서명 1985년 발표된 것으로 이산대수 문제를 이용한 전자서명. 전자서명의 안전성은 이산대수 문제에 기반. ElGamal은 암호화 알고리즘으로는 사용할 수 없고 오직 전자서명만 가능. 서명자 A는 자신의 개인키를 사용하여 메시지에 대해 전자서명
③ Schnorr 전자서명 IC 카드에 적합한 전자서명 방식. ElGamal을 기반으로 하는 방식으로 서명의 크기를 줄이기 위해 제안
④ DSS
(Digital Signature Standard)		 미국 전자서명 표준으로, ElGamal 전자서명을 개량한 것. RSA와 다르게 암호화 교환과 암호화는 지원하지 않고 오직 전자서명만 지원. ElGamal 전자서명 방식으로 서명 검증을 개량한 것으로 서명검증의 부하를 줄이기 위해서 계산량을 줄인 방식
⑤ KCDSA
(Korea Certificate-based Digital Signature Algorithm)		 1996년에 개발된 국내 전자서명 표준으로 국내 전자서명법을 준용하여 인감과 같은 법적 효력을 가짐. 1998년 한국정보통신기술 협회에서 국가 표준으로 제정. 이산대수를 기반으로 하는 전자서명 알고리즘
⑥ ECC 전자서명
(Elliptic Curve Digital Scheme)		 ECDSA로 타원곡선을 기반으로 하는 전자서명. 타원곡선 알고리즘은 짧은 키를 사용하기 때문에 전자서명을 짧은 시간 내에 생성이 가능
PKI (Public Key Infrastructure) 필기
구성요소주요 기능

블록 암호화 기법 필기+실기
모드특징
ECB
(Electronic Code Book)		 가장 단순한 모드. 평문을 일정한 블록단위로 순차적으로 암호화. DES는 64비트씩, AES는 128비트씩 블록을 나눔. 각각의 블록은 독립적이므로 특정 블록의 에러가 다른 블록에 영향을 주지 않음. 같은 평문 블록은 항상 같은 암호문 → 패턴 분석 가능
CBC
(Cipher Block Chaining)		 ECB 모드와 동일한 방법으로 평문을 일정한 블록단위로 나눔. 최초 키의 생성 버퍼로 IV(Initialization Vector)가 사용되어 첫 번째 블록과 XOR 연산을 통해 암호화. IV는 나뉘어진 일정한 블록 중 하나가 되거나 두 번째 위 블록과 길이가 같은 임의의 값이 될 수 있음. 두 번째 블록부터는 첫 번째 블록의 암호화된 블록과 XOR 연산을 하여 암호화
CFB
(Cipher FeedBack)		 ECB, CBC 모드와는 달리 평문과 암호문의 길이가 동일. 패딩을 추가하지 않고 블록단위 암호화를 스트림 암호화 방식으로 구성하여 비트단위로 암호화를 수행
OFB
(Output FeedBack)		 CFB 모드도 평문과 암호문의 길이가 동일. CFB 모드와 동일하게 패딩을 추가하지 않고 블록단위 암호화를 스트림 암호화 방식으로 구성. 암호화 함수는 키의 생성 시에만 사용되어 암호화와 복호화의 방법이 동일. 암호문을 다시 암호화하면 평문이 나옴. 최초 키의 생성 버퍼로 IV가 사용
CTR
(CounTeR)		 평문 블록과 키 스트림을 XOR 연산하여 암호문을 만듦. 키 스트림은 매 암호화 시 1씩 증가하는 카운터를 암호화한 비트열. 암호화와 복호화 방법이 동일하므로 구현이 간단하고 블록의 암호화 순서가 임의의 순서가 될 수 있어 병렬로 처리 가능

암호화 공격기법

암호화 공격기법 필기
공격 방법설명
암호문 단독 공격
(COA, Ciphertext only Attack)		 공격자는 암호문만을 가지고 공격. 암호문으로부터 평문이나 암호키를 찾아내는 방법으로 통계적 성질과 문장의 특성 등을 추정하여 해독
알려진 평문 공격
(KPA, Known Plaintext Attack)		 공격자는 약간의 평문에 대응하는 암호문을 가지고 있는 상태에서 나머지 암호문에 대한 공격을 하는 방법. 이미 입수한 암호문의 관계를 이용하여 새로운 암호문을 해독하는 방법
선택 평문 공격
(CPA, Chosen Plaintext Attack)		 평문을 선택하면 대응되는 암호문을 얻을 수 있는 상황에서의 공격. 공격자가 사용된 암호기에 접근할 수 있을 때 사용하는 공격방법
선택 암호문 공격
(CCA, Chosen Ciphertext Attack)		 암호문을 선택하면 대응되는 평문을 얻을 수 있는 상태에서의 공격. 가장 강력한 공격 방법
🧠 암기 순서 (공격 강도 약 → 강)
암호문 단독(COA) → 알려진 평문(KPA) → 선택 평문(CPA) → 선택 암호문(CCA)

해시함수
— 조건, 종류, 생일자 공격

해시함수는 임의 길이의 입력을 고정된 길이의 출력으로 변환해. 단방향이라서 출력값으로 원본을 복원할 수 없어. 패스워드 저장, 무결성 검증, 전자서명에 핵심적으로 사용돼.
해시함수의 조건 필기
조건설명
압축 임의의 길이의 평문을 고정된 길이의 출력값으로 변환
제1역상 저항성
(One Way Function, 선이미지 회피성)		 메시지에서 해시값(Hash code)을 구하는 것은 쉽지만 반대로 역방향 계산이 불가능하기 때문에 해시값에서 원래의 메시지를 구하는 것은 매우 어려움. 주어진 해시값 y로부터 h(x)=y를 만족하는 x 값을 찾는 것이 어려워야 함
제2역상 저항성
(2차 선이미지 회피성)		 어떤 블록 x에 대해서 H(y)=H(x)인 y≠x인 값을 찾는 것이 계산적으로 불가능해야 함. 약한 충돌 회피성
충돌회피
(Collision free, 강한 충돌 회피성)		 다른 문장을 사용하였는데도 동일한 암호문이 나오는 현상을 말함. h(M1) = h(M2)인 서로 다른 M1과 M2를 구하는 것은 계산상 불가능해야 함
해시함수의 종류 필기+실기
종류설명
MDCModification Detection Code. 메시지의 변경 여부를 확인
MACMessage Authentication Code. 메시지 인증을 수행
MD5Rivest가 개발한 것으로 안전성에서 떨어지는 MD4 알고리즘을 수정하여 만든 것. MD5 이전의 512비트 64라운드 후 128비트의 출력 해시값을 생성
SHASecure Hash Algorithm. MD 계열의 알고리즘과는 달리 160비트의 출력 해시값을 생성. 80라운드 수행
SHA-1미국 표준의 메시지 압축 알고리즘으로 160비트의 출력 해시값을 생성. 80라운드 수행
SHA-256입력 블록크기 512비트에 최종 256비트를 출력하는 해시함수. 64라운드 수행
SHA-512입력블록 크기 1,024비트에 80라운드 후 512비트를 출력하는 해시함수
생일자 공격 (Birthday Attacks) 필기
🎂
23명 중에서 같은 생일을 가지는 사람이 두 사람 혹은 그 이상이 있을 확률은 1/2보다 크다는 이론.
이를 근거로 한 해시함수의 최소 비트는 160비트 이상이 되어야 함.
