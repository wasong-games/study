Chapter 01 · POINT 004

네트워크 기초
— 토폴로지, HTTP, Stateless vs Stateful

네트워크 보안을 이해하려면 먼저 네트워크가 어떻게 생겼고, 어떤 프로토콜로 통신하는지 알아야 해. 특히 HTTP는 웹 공격의 기반이 되고, Stateless/Stateful 개념은 세션 하이재킹이나 방화벽 이해에 직결돼.
네트워크 토폴로지 필기
형태구조특징
계층형 (Tree)부모-자식 관계의 트리 구조확장이 쉬움. 상위 노드 장애 시 하위 전체 영향
버스형중앙 통신 회선 하나에 여러 노드 연결구성 간단, 비용 저렴. 회선 장애 시 전체 영향
성형 (Star)중앙 컴퓨터를 중심으로 터미널들이 연결된 중앙 집중식중앙 장비 장애 시 전체 마비. 관리 용이
원형 (Ring)인접한 노드들을 연결하는 단방향 전송토큰 방식으로 순서대로 전송. 한 노드 장애 시 전체 영향
망형 (Mesh)모든 노드들이 상호 연결안정성 최高. 구성 비용 매우 높음
Stateless vs Stateful 필기

웹 공격의 상당수가 세션(상태)을 악용하는 방식이야. Stateless와 Stateful의 차이를 알면 세션 하이재킹, 쿠키 공격, CSRF 같은 공격이 왜 가능한지 이해돼.

구분StatelessStateful
개념연결을 유지하지 않음. 요청마다 독립적연결 정보(세션)를 유지함
예시HTTP, DNS, UDPTCP, FTP, SSH
서버세션 정보 저장 불필요 (서버 부담 없음)세션 정보 저장 필요 (서버 필요)
장점설계 간단, 장애 후 재시작 쉬움, 서버에서 빠르게 처리여러 세션 정보 유지 가능
단점매 요청마다 인증 정보 재전송 필요비교적 처리 느림, 복잡한 구현
💡
HTTP가 Stateless인데 로그인이 유지되는 이유?
쿠키(Cookie)나 세션(Session) 토큰을 사용해서 상태를 흉내내. 서버가 세션 ID를 쿠키에 담아서 클라이언트에게 주고, 이후 요청마다 그 쿠키를 보내서 "나 이미 로그인한 사람이야"라고 알리는 거야.
→ 이 세션 ID를 탈취하면 세션 하이재킹 공격이 가능해.

Chapter 02 · POINT 004

TCP/IP
— 3-Way / 4-Way 핸드셰이크, Control Flags

TCP는 신뢰성 있는 연결을 보장하기 위해 연결 시작과 종료를 정해진 절차대로 해. 이 절차를 이해하면 SYN Flooding, 세션 하이재킹 같은 공격이 어떻게 이 절차를 악용하는지 바로 연결돼.
TCP 3-Way Handshaking — 연결 수립 필기+실기

TCP 연결을 맺으려면 클라이언트와 서버가 3번의 메시지를 주고받아. 이 과정에서 Sequence Number를 서로 교환해서 이후 패킷 순서를 추적할 수 있게 해.

단계방향Flag내용
(1) SYN클라이언트 → 서버SYN=1 클라이언트가 연결 요청. Seq Num=1000 (임의 생성). "나 연결하고 싶어, 내 번호는 1000이야"
(2) SYN+ACK서버 → 클라이언트SYN=1, ACK=1 서버가 수락 + 자신의 Seq 번호 전송. Seq Num=2000, ACK Num=1001 (클라이언트 Seq+1). "좋아, 1001번부터 보내줘. 내 번호는 2000이야"
(3) ACK클라이언트 → 서버ACK=1 클라이언트가 최종 확인. ACK Num=2001 (서버 Seq+1). "알았어, 2001번부터 보내줘" → 연결 수립(ESTABLISHED)
🚨
SYN Flooding 공격이 이 과정을 어떻게 악용하나?
공격자가 위조된 IP로 SYN 패킷만 대량 전송 → 서버는 SYN+ACK를 보내고 ACK를 기다리는 반열린 연결(Half-Open) 상태로 대기
→ 서버의 연결 대기 큐(Backlog Queue)가 꽉 참 → 정상 사용자의 연결 요청 처리 불가 → 서비스 거부(DoS)
TCP 4-Way Handshaking — 연결 종료 필기+실기

연결을 끊을 때는 4번의 메시지가 필요해. 양쪽 모두 "나는 다 보냈어"라고 각자 FIN을 보내야 하기 때문이야. 3-way와 달리 SYN+ACK를 한 번에 못 하는 이유는, 서버가 아직 보낼 데이터가 남아있을 수 있기 때문이야.

단계방향Flag상태
① FIN클라이언트 → 서버FIN클라이언트: FIN_WAIT1. "나 다 보냈어, 끊을게"
② ACK서버 → 클라이언트ACK서버: CLOSE_WAIT. 클라이언트: FIN_WAIT2. "알겠어, 근데 나 아직 보낼 거 있어"
③ FIN서버 → 클라이언트FIN서버: LAST_ACK. "이제 나도 다 보냈어, 끊을게"
④ ACK클라이언트 → 서버ACK클라이언트: TIME_WAIT → CLOSED. 서버: CLOSED
💡
TIME_WAIT 상태란?
클라이언트가 마지막 ACK를 보낸 후 바로 연결을 닫지 않고 잠시 기다리는 상태야. 마지막 ACK가 유실됐을 때 서버가 FIN을 재전송하면 다시 ACK를 보내기 위해서야. 일반적으로 2MSL(Maximum Segment Lifetime) 시간 동안 유지.

Flag이름역할

🔍
포트 스캐닝에서 Control Flags 활용
• FIN, NULL, XMAS 스캔: 비정상적인 Flag 조합을 보내서 포트 상태 파악
• 포트가 열려있으면 → 응답 없음 (RFC 793)
• 포트가 닫혀있으면 → RST 응답
→ 방화벽이 SYN 스캔은 막더라도 이 스캔들은 통과할 수 있어
Chapter 03 · POINT 005

네트워크 기반 공격
— SYN Flooding, Smurfing, Teardrop, Land

네트워크 프로토콜의 설계 허점을 이용한 공격들이야. 각 공격이 어떤 프로토콜의 어떤 특성을 악용하는지 연결해서 이해하면 암기가 아니라 논리적으로 기억돼.
TCP SYN Flooding 필기+실기

TCP 3-Way Handshake의 허점을 이용한 DoS 공격이야. 서버가 SYN을 받으면 반드시 ACK를 기다리는 상태로 자원을 할당하는데, 이걸 악용해.

💥
공격 원리
① 공격자가 위조된(가짜) IP 주소로 SYN 패킷을 대량 전송
② 서버는 SYN+ACK를 보내고 ACK를 기다리는 Half-Open 상태로 연결 대기 큐(Backlog Queue)에 등록
③ 가짜 IP라서 ACK는 절대 오지 않음 → 큐가 꽉 참
④ 정상 사용자의 SYN 패킷은 큐에 들어오지 못해 → 서비스 불가

대응 방법
• Backlog Queue 크기 늘리기: sysctl -w net.ipv4.tcp_max_syn_backlog=1024
• SYN Cookie 사용: ACK 오기 전까지 큐에 넣지 않고 쿠키로 검증
• 방화벽에서 동일 IP의 SYN 패킷 임계치 설정
UDP Flooding 필기+실기

공격자의 IP를 피해자의 IP로 변경(스푸핑)하여 UDP 패킷을 지속적으로 전송하면, ICMP Destination Unreachable 메시지가 발생하고 해당 메시지가 피해자에게 수신돼 대역폭을 소모시켜.

NTP 증폭 공격 필기+실기

NTP(Network Time Protocol)는 시간 동기화 서버들이 내부 서버들의 시간을 동기화시키는 프로토콜이야. #ntp -n -c monlist (점검대상 NTP서버 IP)를 전송하면 응답으로 전송되는 서버 목록을 피해자에게 전송하게 해. 요청 패킷보다 응답 패킷이 수백 배 커서 증폭 효과가 발생해.

Smurfing 공격 필기+실기

IP의 특성인 Broadcast 주소와 ICMP 패킷을 이용한 공격이야.

💥
공격 원리
① 공격자가 출발지 IP를 피해자 IP로 위조
② 목적지를 브로드캐스트 주소로 설정한 ICMP Echo Request 전송
③ 해당 네트워크의 모든 호스트가 ICMP Echo Reply를 피해자에게 전송
④ 피해자는 수많은 ICMP 응답으로 대역폭 소모 → 서비스 불가

대응 방법
• 유닉스: ndd -set /dev/ip ip_forward_directed_broadcasts 0
• 리눅스: sysctl -w net.ipv4.icmp_echo_ignore_broadcast=1
→ Directed Broadcast를 차단하면 방어 가능
Teardrop 공격의 종류 필기

IP 패킷은 크기가 크면 여러 조각(Fragment)으로 나뉘어 전송되고 수신측에서 재조합돼. Teardrop 계열 공격은 이 IP 단편화(Fragmentation) 처리의 허점을 이용해.

공격 종류원리
Tiny Fragment 최초의 Fragment를 아주 작게 만들어서 네트워크 침입탐지 시스템이나 패킷 필터링 장비를 우회하는 것이 목적
Fragment Overlap Tiny Fragment보다 더욱 정교한 방법. IDS의 Fragment 처리 방법과 패킷 필터링의 재조합과 overwrite 처리를 이용. IDS를 속이는 데 목적
Ping of Death Ping을 이용해 ICMP 패킷을 규정된 길이 이상으로 IP 패킷을 전송. 수신 측 OS에서 처리하지 못하게 하여 시스템을 마비시키는 공격
Tear Drop Fragment 재조합 과정의 취약점을 이용한 공격. 목표 시스템 정지나 재부팅을 유발. TCP Header 부분의 offset field 값이 중첩되는 데이터 패킷을 대상 시스템에 전송
Land Attack 필기
💥
IP Header를 변조하여 인위적으로 송신자 IP 주소 및 Port를 수신자의 IP 주소와 Port 주소로 동일하게 설정하여 트래픽을 전송하는 공격기법이야.

패킷의 출발지와 목적지가 동일해서 시스템이 자기 자신에게 계속 응답하다가 과부하 상태가 됨.

HTTP DDoS 공격과 세션 하이재킹

HTTP DDoS 공격 필기+실기
공격 종류원리
Slow HTTP POST
(RUDY 공격)		 HTTP의 POST 지시자를 사용해 서버에게 전달할 대량의 데이터를 장기간에 걸쳐 분할 전송. POST 데이터가 모두 수신되지 않으면 연결을 장시간 유지하게 됨 → 다수 연결로 서버 자원 소진
Slow HTTP Read DoS 공격자가 웹 서버와 TCP 연결 시 TCP 윈도우 크기 및 데이터 처리율을 감소시켜 HTTP 데이터를 송신하여 웹 서버가 정상적으로 응답하지 못하게 하는 DoS/DDoS 기법. TCP 윈도우 크기를 감소시키면 서버는 정상상태로 회복될 때까지 대기상태에 빠져서 부하를 유발
Slow HTTP Header DoS
(Slowloris)		 HTTP Header를 비정상적으로 조작하여 웹 서버가 헤더 정보를 구분할 수 없도록 만듦. 웹 서버가 아직 HTTP Header의 정보가 모두 전달되지 않은 것으로 판단하게 만들어 연결을 장시간 유지하게 함.
웹 서버는 클라이언트로부터 요청이 끝나지 않은 것으로 판단되기 때문에 웹 로그에 기록되지 않는다는 특징이 있음 → 탐지 어려움
세션 하이재킹 필기+실기
🎯
인증을 위한 모든 검증을 우회하는 공격이야. TCP를 이용해서 통신하고 있을 때 RST(Reset) 패킷을 보내 일시적으로 TCP 세션을 끊고, Sequence Number를 새로 생성하여 세션을 빼앗고 인증을 회피해.

공격 흐름
① 공격자가 클라이언트-서버 간 통신을 스니핑으로 감청
② 현재 Sequence Number 파악
③ 클라이언트에게 RST 패킷 전송 → 클라이언트 연결 끊김
④ 공격자가 올바른 Seq Number로 서버에 패킷 전송 → 서버는 공격자를 클라이언트로 인식

포트 스캐닝과 스니핑
— 정찰 단계의 핵심 기법

공격자가 본격적인 공격 전에 목표 시스템의 어떤 포트가 열려있는지, 어떤 서비스가 동작하는지 파악하는 정찰 단계야. 사이버 킬체인 1단계(정찰)에 해당해.
포트 스캐닝 필기+실기
상황스캐닝 기법원리
포트 Open 시 응답 없음 UDP, FIN, X-MAS(FIN+PSH+URG), NULL 스캔 열린 포트는 이상한 패킷에 응답하지 않음 (RFC 793)
포트 Close 시 RST로 응답 FIN, X-MAS(FIN+PSH+URG), NULL 스캔 닫힌 포트는 RST 패킷으로 응답
UDP스캔 포트 Close 시 UDP 스캔 ICMP Destination Unreachable로 응답 → 닫힌 포트 확인
TCP스캔 포트 Open 시 TCP SYN 스캔 (Half-Open) SYN+ACK로 응답 → 열린 포트 확인. 연결 완성 안 해서 로그에 안 남을 수 있음
TCP스캔 포트 Close 시 TCP SYN 스캔 RST+ACK로 응답
스니핑(수동적 공격) 필기+실기

스니핑 도구를 실행시키면 기본적으로 Normal Mode(정규모드)로 실행돼. Normal Mode는 자신의 컴퓨터에 전송되는 패킷만 수신하고 자신과 관련 없는 패킷은 삭제(Drop)해.

🎧
네트워크의 모든 패킷을 모니터링하려면
→ Promiscuous Mode(무차별 모드)로 설정하고 스니핑 실행

• 무차별 모드 설정: ifconfig eth0 promisc
• 무차별 모드 해제: ifconfig eth0 -promisc

스니핑 탐지 방법
• ifconfig로 PROMISC 플래그 확인
• ARP 스푸핑 탐지 도구 사용
• Ping을 이용한 응답시간 비교 (스니퍼가 있으면 살짝 느림)

옵션설명

옵션설명

nocase대문자와 소문자를 구분하지 않음

flagsTCP의 Control Flags를 탐지
itypeICMP 프로토콜의 타입 값을 탐지
threshold동일한 패킷이 설정한 시간 내에서 일정한 수가 탐지되면 경고를 발생시킴

모드설명

무선 LAN 보안
— WEP, WPA, WPA2, IEEE 802.11

무선 네트워크는 유선과 달리 공기 중에 신호가 퍼지기 때문에 누구나 신호를 수신할 수 있어. 그래서 암호화가 더 중요하고, 암호화 방식이 약하면 쉽게 해독될 수 있어.
무선 LAN 보안 프로토콜 필기+실기
프로토콜암호화특징 및 취약점
WEP RC4 대칭형 암호화 알고리즘, 40Bit 키 24Bit 길이의 초기화 벡터(IV, Initialization Vector)를 사용. 무작위 공격에 취약. IV가 짧아서 재사용되며 패턴 분석으로 키 해독 가능. 현재 사용 금지 권고
WPA
(IEEE 802.1x/EAP)		 128Bit 동적 암호화 및 복호화(RC4 사용) TKIP(Temporal Key Integrity Protocol) 방식으로 사용자, 네트워크 세션, 전송 프레임별로 키를 동적으로 생성. 상호인증을 위해 EAP를 지원. WEP보다 강력하지만 여전히 취약점 존재
WPA2

SNMP는 네트워크 관리를 위해서 정의된 프로토콜로 네트워크 장비 관리 및 감시를 위해 (ㄱ)상에 정의된 응용 계층 프로토콜이야. SNMP Manager와 SNMP Agent 간에 특정 정보를 주고받는 것이 네트워크 관리의 기본이며 이러한 관리 정보를 (ㄴ)MIB라고 한다. SNMP Community String은 SNMP에서 일종의 패스워드 역할을 수행하는 것으로 SNMP Community 이름이 (ㄷ)Public, (ㄹ)Private인 경우 보안에 취약하다.

SDN vs NFV 필기

네트워크의 효율성과 QoS를 효율적으로 관리하기 위해서 네트워크 장치의 제어부와 전송부를 분리하는 개념으로 중앙에서 여러 네트워크를 관리할 수 있는 것 → SDN

기술설명
SDN (Software Defined Networking) 네트워크 장치의 제어부와 전송부를 분리하는 것으로 여러 네트워크를 논리적으로 관리하는 관리 기술이다. 소프트웨어를 사용해서 네트워크를 효율적으로 관리한다
NFV (Network Function Virtualization) 네트워크 기능을 분리하는 것으로 물리적 네트워크 장치에서 소프트웨어 네트워크 기능(방화벽, 로드밸런싱, 포워딩 등)을 가상화 기술을 이용하여 제공한다
TMS (Threat Management System) 위협 관리 시스템
CDR (Content Disarm & Reconstruction) 콘텐츠 무해화 기술 — 파일에서 위협 요소를 제거하고 재구성
APT (Advanced Persistent Threat) — 지능형 지속 공격 필기+실기
🎯
APT(Advanced Persistent Threat)는 개인 및 정부 기관, 기업을 대상으로 지속적인 해킹 공격을 수행해서 개인정보나 중요 데이터를 유출하는 공격 형태로 방어하기가 어렵다.

APT 특징 (기출 오답 포인트)
① 개인 및 정부 기관을 대상으로 DDoS와 같은 네트워크 가용성을 공격한다 ❌
→ APT는 DDoS가 아니라 데이터 유출과 정보 탈취가 목적
② 지능형 공격에 대해서 접점 구간에 방화벽을 설치해서 대응 가능하다 ❌
→ 방화벽만으로는 대응 어려움
③ 특정 타깃을 대상으로 개인정보나 중요 데이터를 유출하며 방어가 어렵다 ✅
④ 계획적인 접근으로 악성 이메일을 사용해서만 공격한다 ❌
→ 이메일 외에도 다양한 벡터 사용
스니핑 무차별 모드 — 대응 방법 필기+실기
🎧
스니핑(Sniffing)의 무차별 모드(Promiscuous)는 자신에게 전송되는 MAC 주소뿐만 아니라 다른 MAC 주소까지 모두 수신하여 전송되는 패킷을 훔쳐볼 수 있다.
따라서 무차별 모드를 활용한 스니핑에 대비하기 위해서는 전송되는 데이터를 암호화해야 한다.

빈칸 (ㄱ) = MAC, (ㄴ) = 암호화 → 정답 ② (ㄱ) MAC, (ㄴ) 암호화
OSI 7계층 — Transport 계층 필기+실기

End-to-End 구간에서 신뢰성 있는 데이터 송수신을 하고 흐름 제어와 혼잡 제어를 수행하는 계층 → Transport 계층 (전송 계층)

💡
전송 계층(Transport Layer)은 TCP와 UDP가 있으며 TCP는 신뢰성 있는 데이터 전송과 흐름 제어, 에러 처리, 혼잡 제어 기능을 제공한다.

• Network 계층: IP 주소 기반 라우팅
• Data Link 계층: MAC 주소 기반 프레임 전송
• Transport 계층: 포트 번호, 흐름제어, 혼잡제어, End-to-End ← 정답

Spoofing 종류 — UDP Spoofing은 없음 필기+실기

Spoofing은 결과 값이 공격자에 의해서 변조된 공격이야. "UDP Spoofing"은 Spoofing의 종류가 아님!

Slow HTTP Post — Content_Length 활용 필기+실기
🐌
처음 패킷은 Content_Length를 크게 해서 전송하고 다음 패킷부터 Content_Length를 1바이트씩 작게 만들어 천천히 전송하는 공격 기법이야. 천천히 데이터를 전송하여 시스템에 더 많은 부하를 유발한다.

Slow HTTP Post는 공격자가 POST 메시지의 Content-Length 값을 크게 설정한 후에 소량의 데이터를 천천히 웹 서버로 전송하는 DDoS 공격 기법이다.
IDS 오용탐지 — 변경된 공격 탐지 불가 필기+실기
⚠️

오용탐지의 올바른 특징:
① 공격자 측면에서 오용탐지는 다양한 방법으로 우회 가능성이 존재한다 ✅
② 오용탐지를 최소화하기 위해서는 세밀한 패턴 정의가 필요하다 ✅
④ 오용탐지는 잘못된 탐지 가능성이 낮다 ✅ (False Positive가 낮음)
SOAR (Security Orchestration, Automation and Response) 필기+실기

🤖

허니팟 vs 허니넷 필기+실기

"네트워크의 침입 발생 시에 탐지하고 즉시 대응한다" → 이건 허니팟에 대한 올바르지 않은 설명이야.

🍯
허니팟(Honeypot): 비정상 행위를 위한 함정이고
허니넷(Honeynet): 다수의 허니팟으로 구성된 네트워크이다.

허니팟으로 공격자를 유인해서 비정상 행위를 탐지하고 Zero Day Attack에 대응할 수 있다.

올바르지 않은 설명 (기출)
① "네트워크의 침입 발생 시에 탐지하고 즉시 대응한다" ❌
→ 허니팟은 즉시 대응이 아니라 공격자를 유인해서 행동 분석하는 것
EternalBlue — WannaCry, LockBit 필기
💥
이터널블루(EternalBlue)는 SMB 원격 코드 실행 취약점을 가지고 있는 것으로 이터널블루로 볼티모어시에 대해서 랜섬웨어를 수행했고 이때 사용된 랜섬웨어가 로빈후드이다.

로빈후드 랜섬웨어는 최근 정상적으로 서명된 드라이버들을 사용해 보안 도구를 삭제하고 파일을 암호화시키는 방법이다.

SNMP v3 → 버퍼오버플로우/무차별대입/Injection/Replay/세션키 스니핑 대응 가능
비대면 업무환경 보안 필기+실기
⚠️
"원격근무지에서 VPN을 사용하여 연결하면 다중 인증 기능을 사용하지 않아도 된다" ❌ → 오답!
원격 근무를 통해서 기업 정보시스템에 접근할 때는 2-Factor 인증을 수행해야 한다.

올바른 비대면 업무환경 보안:
① 원격근무를 위해서 개인정보 PC에 대한 보호조치가 필요하다 ✅
③ 인증 시에 2-Factor 인증을 수행하고 전송되는 데이터는 암호화를 해야 한다 ✅
④ 재택에서 기업 정보시스템 접근 시에 원격접근계정에 대해서 접근 통제를 수행해야 한다 ✅
TCP SYN Flooding — netstat 화면으로 판별 필기+실기
🖥️
netstat 화면에서 같은 Local Address에 여러 Foreign Address가 모두 "TCP RECEIVED" 상태인 경우
→ TCP SYN Flooding 공격이야.

TCP Received는 TCP SYN 메시지에 대해서 서버가 SYN+ACK를 전송했다는 의미다. 따라서 본 공격은 TCP SYN Flooding 공격이다.

• FTP 바운스 Attack: FTP PORT 명령 악용
